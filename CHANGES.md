# تعديلات نظام الفوترة

## التاريخ: 2026-06-17

---

## ملخص التغييرات

تم تعديل نظام إنشاء دورات الفوترة بحيث تُنشأ الفواتير **محسوبة بالكامل** مباشرة عند إنشاء دورة جديدة، بدلاً من إنشائها فارغة والانتظار لإدخال القراءات يدوياً.

---

## التغييرات المنفذة

### 1. تعديل API إنشاء دورة الفوترة

**الملف:** `src/app/api/billing/route.ts`

#### قبل التعديل:
- عند إنشاء دورة فوترة، تُنشأ فواتير مسودة بـ:
  - `consumption = 0`
  - `currentReading = previousReading`
  - `totalAmount = workUnitsTotal` (فقط رسوم وحدات العمل)
- يجب على المستخدم إدخال القراءات يدوياً ثم الحفظ

#### بعد التعديل:
- عند إنشاء دورة فوترة، تُنشأ الفواتير **محسوبة بالكامل**:
  - `currentReading = previousReading + workUnits`
  - `consumption = workUnits` (الاستهلاك الافتراضي = وحدات العمل)
  - يحسب الشريحة الأولى (0-4 وحدات × 700 ريال)
  - يحسب الشريحة الثانية (فوق 4 وحدات × 1000 ريال)
  - `totalAmount = workUnitsTotal + tier1Cost + tier2Cost`

#### الكود المضاف:
```typescript
import { calculateBill } from '@/lib/billing';

// Inside POST handler:
const currentReading = prevReading + workUnits;

const calc = calculateBill({
  workUnits,
  previousReading: prevReading,
  currentReading,
  workUnitPrice,
  tier1Limit,
  tier1Price,
  tier2Price,
});

await tx.bill.create({
  data: {
    // ...
    currentReading,
    consumption: calc.consumption,
    workUnitsTotal: calc.workUnitsTotal,
    tier1Units: calc.tier1Units,
    tier1Cost: calc.tier1Cost,
    tier2Units: calc.tier2Units,
    tier2Cost: calc.tier2Cost,
    totalAmount: calc.totalAmount,
    // ...
  },
});
```

### 2. تعديل صفحة الفوترة

**الملف:** `src/app/billing/page.tsx`

#### التغييرات:
- إضافة أعمدة تفصيلية في جدول الفواتير:
  - **رسوم الوحدات**: `workUnits × 2000 ريال`
  - **الشريحة الأولى**: `tier1Units × 700 ريال`
  - **الشريحة الثانية**: `tier2Units × 1000 ريال`
- الفواتير تظهر محسوبة مباشرة عند اختيار الدورة
- لا يزال بإمكان المستخدم تعديل القراءة يدوياً وحفظ التعديلات
- تغيير نص زر الحفظ من "حفظ جميع القراءات" إلى "حفظ التعديلات"

// في كل صف:

### 3. حساب الفاتورة (محرك الحساب)

**الملف:** `src/lib/billing.ts` (لم يتغير - يُستخدم كما هو)

#### معادلة الحساب:
```
الاستهلاك = القراءة الحالية - القراءة السابقة
رسوم وحدات العمل = عدد الوحدات × 2000 ريال
الشريحة الأولى = min(الاستهلاك، 4) × 700 ريال
الشريحة الثانية = max(الاستهلاك - 4، 0) × 1000 ريال
المبلغ الإجمالي = رسوم وحدات العمل + الشريحة الأولى + الشريحة الثانية
```

---

## مثال على الحساب

**المدخلات:**
- وحدات العمل: 1
- القراءة السابقة: 34.4
- القراءة الحالية: 35.4 (34.4 + 1)

**النتائج:**
- الاستهلاك: 1.0 م³
- رسوم وحدات العمل: 1 × 2000 = 2,000 ريال
- الشريحة الأولى: 1.0 × 700 = 700 ريال
- الشريحة الثانية: 0 × 1000 = 0 ريال
- **المبلغ الإجمالي: 2,700 ريال**

---

## مثال آخر (استهلاك أعلى)

**المدخلات:**
- وحدات العمل: 1
- القراءة السابقة: 34.4
- القراءة الحالية: 47.1

**النتائج:**
- الاستهلاك: 12.7 م³
- رسوم وحدات العمل: 1 × 2000 = 2,000 ريال
- الشريحة الأولى: 4.0 × 700 = 2,800 ريال
- الشريحة الثانية: 8.7 × 1000 = 8,700 ريال
- **المبلغ الإجمالي: 13,500 ريال** ✅ (مطابق للاختبار)
